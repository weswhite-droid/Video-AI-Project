4 Functional Requirements ▼
4a · Alarm Monitoring & Web Application Portal
# Requirement Detail Status
AM-01 Automated root cause analysis for all security events Upon receiving a signal from the Lenel OnGuard Alarm Monitoring system, the Cobalt Video Analytics platform shall automatically ingest, analyze, and categorize the underlying cause of the event while notating analysis within the alarm notes. Completed
AM-02 AI-based nuisance alarm filtering before escalation All alarms must be analyzed to filter out false alarms prior to escalation to GO Center. Completed
AM-03 Real-time escalation to GO Center within 30–60 seconds of genuine threat detection — Completed
AM-04 Human review required for all legitimate violation detections (tailgate, unauthorized entry) Cases generated must NOT auto-close without human disposition. Currently only 3–5% of detected violations receive human investigation — this gap must be closed before Phase 2. ⚠ At Risk
AM-05 Web application portal dashboard must be kept "clean" Dashboard must contain only alarms escalated to GO Center. All alarms not being escalated to GOC must be acknowledged/archived. In Progress
4b · Violation / Incident Detection & Response
Violation Type Workflow Status
Tailgating / Piggybacking
Defined — view workflow ▾
  • Tailgating event → Office Agent → local Slack channel; Office Agent creates Service Cloud Case (Non-Compliance Tracking)
  • Vendor identifies tailgator & tailgatee using ACS → pings Office Agent → updates case and Slack
  • Office Agent sends Slack message to tailgatee (1st through door): Do you know the tailgator? (Yes / No)
    • YES: tailgatee enters name → reminders sent to both parties → case auto-closed in Service Cloud
    • NO: Office Agent notifies SecOps + GOC via Slack → guard dispatched → human handles final resolution and case closure
 In Progress
Weapons Detection
